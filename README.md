Summarize the project and what problem it was solving.

This project focused on building a smart thermostat prototype using a Raspberry Pi, sensors, buttons, LEDs, and an LCD display. 
The goal was to create a working system that could monitor room temperature, allow the user to change the set temperature, and switch between heating, cooling, and off modes. 
It solved the problem of showing how hardware and software can work together to automate temperature control in a simple embedded system.

What did you do particularly well?

One thing I did particularly well was troubleshooting problems with the circuit and wiring. 
During the project, I had to test buttons, check connections, and make corrections when parts of the circuit did not work as expected.
I also stayed patient and kept testing until the thermostat responded correctly. This helped me better understand how hardware and software interact.


Where could you improve?

One area where I could improve is planning and organizing the circuit layout before building it. At times, wiring became confusing and took extra time to fix. 
In future projects, I would spend more time labeling connections, keeping wires organized, and documenting changes as I go. 
This would make troubleshooting easier and save time later.


What tools and/or resources are you adding to your support network?

The tools and resources I am adding to my support network include Python programming practice, Raspberry Pi documentation, GPIO reference guides, and online forums for troubleshooting electronics projects.
I also learned how valuable clear instructions, diagrams, and testing scripts can be when solving technical issues.

What skills from this project will be particularly transferable to other projects and/or course work?

The skills from this project that will transfer to other coursework include problem-solving, debugging, coding in Python, reading technical instructions, and working with hardware components.
I also improved my ability to test systems step by step, which is useful in programming, networking, and future engineering projects.


How did you make this project maintainable, readable, and adaptable?

I made this project maintainable, readable, and adaptable by using organized code, comments, and clear variable names in the program. 
I also kept the wiring consistent and tested each component individually before combining everything together. 
This makes it easier to update the code, replace parts, or expand the project with new features in the future.
